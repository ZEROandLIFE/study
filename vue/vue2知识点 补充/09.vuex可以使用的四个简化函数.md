`mapState`、`mapGetters`、`mapMutations` 和 `mapActions` 是四个核心辅助函数，用于简化组件与 Vuex Store 的交互。它们分别用于映射 **State**、**Getters**、**Mutations** 和 **Actions** 到组件的计算属性或方法中，避免手动编写重复代码。

---

## **1. `mapState`：映射 State 到计算属性**
### **作用**
将 Vuex Store 中的 **State** 映射到组件的 **计算属性（computed）** 中，方便直接访问 State 数据。

### **基本用法**
#### **（1）数组形式（直接映射同名属性）**
```javascript
import { mapState } from 'vuex';

export default {
  computed: {
    ...mapState(['count', 'user']), // 将 this.count 映射为 this.$store.state.count
  },
};
```
在模板中直接使用：
```html
<p>当前计数：{{ count }}</p>
```

#### **（2）对象形式（自定义计算属性名）**
```javascript
computed: {
  ...mapState({
    localCount: 'count', // 将 this.localCount 映射为 this.$store.state.count
    currentUser: 'user',
  }),
},
```
模板中使用：
```html
<p>本地计数：{{ localCount }}</p>
```

#### **（3）结合本地计算属性**
如果需要混合本地计算属性和 Vuex State：
```javascript
computed: {
  ...mapState(['count']),
  doubleCount() {
    return this.count * 2; // 本地计算属性
  },
},
```

---

## **2. `mapGetters`：映射 Getters 到计算属性**
### **作用**
将 Vuex Store 中的 **Getters** 映射到组件的 **计算属性（computed）** 中，方便直接访问派生数据（如过滤、计算后的 State）。

### **基本用法**
#### **（1）数组形式（直接映射同名 Getter）**
```javascript
import { mapGetters } from 'vuex';

export default {
  computed: {
    ...mapGetters(['doubleCount', 'filteredUsers']), // 映射 Getters
  },
};
```
模板中使用：
```html
<p>双倍计数：{{ doubleCount }}</p>
```

#### **（2）对象形式（自定义计算属性名）**
```javascript
computed: {
  ...mapGetters({
    localDoubleCount: 'doubleCount', // 映射 Getter 并重命名
    userList: 'filteredUsers',
  }),
},
```
模板中使用：
```html
<p>本地双倍计数：{{ localDoubleCount }}</p>
```

---

## **3. `mapMutations`：映射 Mutations 到组件方法**
### **作用**
将 Vuex Store 中的 **Mutations** 映射到组件的 **methods** 中，方便直接提交（`commit`） Mutation 来修改 State。

### **基本用法**
#### **（1）数组形式（直接映射同名方法）**
```javascript
import { mapMutations } from 'vuex';

export default {
  methods: {
    ...mapMutations(['increment', 'decrement']), // 映射 Mutation
  },
};
```
在方法中调用：
```javascript
methods: {
  handleClick() {
    this.increment(2); // 提交 `increment` Mutation 并传递参数
  },
},
```

#### **（2）对象形式（自定义方法名）**
```javascript
methods: {
  ...mapMutations({
    add: 'increment', // 将 this.add() 映射为 this.$store.commit('increment')
    remove: 'decrement',
  }),
},
```
调用方式：
```javascript
methods: {
  handleClick() {
    this.add(5); // 提交 `increment` Mutation 并传递参数
  },
},
```

#### **（3）直接提交 Mutation（不映射）**
如果不映射，也可以手动提交：
```javascript
methods: {
  handleClick() {
    this.$store.commit('increment', 2); // 直接提交 Mutation
  },
},
```

---

## **4. `mapActions`：映射 Actions 到组件方法**
### **作用**
将 Vuex Store 中的 **Actions** 映射到组件的 **methods** 中，方便直接分发（`dispatch`） Action（通常用于异步操作）。

### **基本用法**
#### **（1）数组形式（直接映射同名方法）**
```javascript
import { mapActions } from 'vuex';

export default {
  methods: {
    ...mapActions(['fetchData', 'asyncIncrement']), // 映射 Action
  },
};
```
调用方式：
```javascript
methods: {
  handleClick() {
    this.fetchData(); // 分发 `fetchData` Action
  },
},
```

#### **（2）对象形式（自定义方法名）**
```javascript
methods: {
  ...mapActions({
    getData: 'fetchData', // 将 this.getData() 映射为 this.$store.dispatch('fetchData')
    increaseAsync: 'asyncIncrement',
  }),
},
```
调用方式：
```javascript
methods: {
  handleClick() {
    this.getData(); // 分发 `fetchData` Action
  },
},
```

#### **（3）传递参数**
如果 Action 需要参数：
```javascript
methods: {
  ...mapActions(['incrementBy']),
  handleClick() {
    this.incrementBy(10); // 分发 `incrementBy` Action 并传递参数
  },
},
```

#### **（4）直接分发 Action（不映射）**
如果不映射，也可以手动分发：
```javascript
methods: {
  handleClick() {
    this.$store.dispatch('fetchData'); // 直接分发 Action
  },
},
```

---

## **5. 综合对比**
| 辅助函数       | 作用                      | 映射目标    | 使用场景                     |
| -------------- | ------------------------- | ----------- | ---------------------------- |
| `mapState`     | 映射 State 到计算属性     | `state`     | 直接访问 Store 中的数据      |
| `mapGetters`   | 映射 Getters 到计算属性   | `getters`   | 访问派生数据（如过滤、计算） |
| `mapMutations` | 映射 Mutations 到组件方法 | `mutations` | 同步修改 State               |
| `mapActions`   | 映射 Actions 到组件方法   | `actions`   | 异步操作或复杂逻辑修改 State |

---

## **6. 模块化中的使用**
如果 Vuex Store 拆分为模块，可以通过 **命名空间** 指定模块：
```javascript
methods: {
  ...mapMutations('user', ['updateProfile']), // 映射 `user` 模块的 `updateProfile` Mutation
  ...mapActions('order', ['fetchOrders']),    // 映射 `order` 模块的 `fetchOrders` Action
},
```
或使用对象形式：
```javascript
methods: {
  ...mapMutations({
    updateUser: 'user/updateProfile', // 映射 `user` 模块的 `updateProfile` Mutation
  }),
  ...mapActions({
    getOrders: 'order/fetchOrders',   // 映射 `order` 模块的 `fetchOrders` Action
  }),
},
```

## **7. 完整示例**
```javascript
import { mapState, mapGetters, mapMutations, mapActions } from 'vuex';

export default {
  computed: {
    ...mapState(['count']), // 映射 State
    ...mapGetters(['doubleCount']), // 映射 Getters
  },
  methods: {
    ...mapMutations(['increment']), // 映射 Mutation
    ...mapActions(['asyncIncrement']), // 映射 Action
    handleSyncClick() {
      this.increment(3); // 提交 Mutation
    },
    handleAsyncClick() {
      this.asyncIncrement(); // 分发 Action
    },
  },
};
```

---

## **总结**
- **`mapState`**：简化 `this.$store.state` 的访问。
- **`mapGetters`**：简化 `this.$store.getters` 的访问。
- **`mapMutations`**：简化 `this.$store.commit` 的调用。
- **`mapActions`**：简化 `this.$store.dispatch` 的调用。

它们可以显著减少样板代码，使组件更简洁，同时保持与 Vuex 的紧密集成。
